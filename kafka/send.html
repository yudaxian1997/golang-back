01.send message

kafka生产者发送消息

    1. ProducerRecord 创建
    2. ProducerRecord 是 Kafka 生产者用来封装待发送消息的一个类，包含以下内容
      • Topic：消息的目标 Topic。
      • Partition：消息的目标分区（如果指定了 key，分区会自动根据 key 计算）。
      • Key 和 Value：消息的键和值，分别用于分区和消息的内容。
    1. 序列化消息
    2. 一旦 ProducerRecord 被创建，生产者会将消息的 key 和 value 序列化成字节流
    1. 根据分区策略选择目标分区
    2. Kafka 生产者会根据消息的 key 选择目标分区。
    3. 具体是通过 key.hashCode() % numPartitions 来计算
    1. 发送到服务器
    2. 网络发送：通过 Socket 连接将消息发送到 Kafka Broker 的对应分区。
    3. ACK：Kafka 可以配置不同的 acks 级别来确认消息是否成功写入：acks=0（不等待确认）、acks=1（等待 Leader 确认）、acks=all（等待所有副本确认）。
    4. 失败重试：如果发送消息失败，生产者会按照一定的重试机制进行重试。

      Kafka 生产者支持三种发送方式，分别是 发送并忘记（Fire-and-forget）、同步发送（Synchronous Send）和 异步发送（Asynchronous Send）
