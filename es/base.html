01.基础操作

基本概念和使用

ElasticSearch: 一个开源的分布式搜索引擎，可以用来实现搜索，日志统计，分析和系统监控 ELK: 是以elasticsearch为核心的技术栈，包括beats，Logstach，kibana，elasticsearch

倒排索引

  • 1.mysql正向索引
    • id title price
    • 1 小米手机 3499
    • 2 华为手机 4999
    • id -> row 这种b+tree就是正向索引
  • 2.es倒排索引
    • term doc_id
    • 手机 1，2
    • 小米 1
    • 华为 2
    • 分词之后的term -> 文档id 就是倒排索引
    • 搜索的时候先对"搜索内容进行分词"，然后根据倒排索引进行查询

es中的常见概念

  • 1.文档: es是面向文档存储的，可以将数据库中的一条数据序列化成json之后存入es
    • {id: "1", "title": "小米手机", "price": 3499}
  • 2.索引（index）: 相同类型的文档的集合
  • 3.Mapping索引中的文档约束，字段约束，类似mysql的表结构
  • 4.DSL: json风格的请求语句

分词器

POST /_analyze
{
    "analyzer": "standard",
    "text": "content"
}
  • 标准分词器对中文不友好，使用ik分词器，支持 stop_word.dic 和 ext.dic
    • ik_smart
    • ik_max_word

es索引操作

  • mapping属性
    • type: 字段的数据类型
      • 字符串: text(可分词的文本) , keyword (精确值，例如: 品牌，国家，ip地址)
      • 数值类型: long integer short double float
      • 布尔类型: boolean
      • 日期类型: date
      • 对象类型: object
    • index: true or false 是否创建倒排索引，默认true
    • analyzer: 使用什么分词器
    • properties: 该字段的子属性
PUT /index_name
{
  "mappings": {
    "properties": {
      "info": {
        "type": "text",
        "analyzer": "ik_smart"
      },
      "email": {
        "type": "text",
        "index": false
      },
      "name": {
        "type": "object",
        "properties": {
          "firstName": {
            "type": "keyword"
          },
          "lastName": {
            "type": "keyword"
          }
        }
      }
    }
  }
}

es文档查询

DSL Query的分类

  • 1.查询所有: match_all
  • 2.全文检索: 利用分词器对输入内容进行分词，然后去匹配倒排索引
    • match_query
    • multi_match_query
  • 3.精确分词: 内容整体进行查询，不进行分词
    • ids rang term
  • 4.复合查询: 将多个查询结合起来
    • bool function_score
GET /index_name/_search
{
    "query": {
        "match_all": {},
        "match": { // 全文检索
            "字段名字": "检索内容"
        },
        "multi_match": { // 多字段全文检索
            "query": "text",
            "字段数组": ["字段1", "字段2"]
        },
        "term": { // 精确查询
            "字段名": {
                "value": "字段值"
            }
        },
        "range": { // 范围查询
            "字段名": {
                "gte": "min",
                "lte": "max"
            }
        },
        // 相关性算分， TF-IDF  BM25
        "function_score": {
            "query": {}
            "functions": [
                "filter": {"term": {"id": 1}},
                "weight": 10
            ],
            // 加权模式 sum avg max min
            "boost_mode": "multiply"
        },
        // must(&&) should(||)  must_not(!) filter 后面这两个不参与算分
        "bool": {
            "must": [
                {"term": {"name": "value"}}
            ],
            "should": [
                {"term": {"name": "value"}}
            ],
            "must_not": [
                {"range": {"price": {"lte": 500}}}
            ],
            "filter": [
                {"range": {"score": {"gte": 45}}}
            ]
        },
        // 排序分页
        "sort": [
            {"字段名": "desc"}
        ],
        from: 999,
        size: 10
    }
}
  • 分页需要注意，ES使用的倒排索引，需要查询所有的数据，再内存中进行逻辑分页
  • 深度分页问题
    • ES是分布式，比如按Price排序后取 from 990 size 10 的数据
    • 会把所有分片的里面的数据都查出来 top1000 再汇总排序，取10条
    • 由于内存排序结果集上限不能超过10000
    • 如果需要超过1w的结果集，使用search after

es聚合

  • 1.桶(Bucket)聚合
    • Term聚合按照字段值聚合 Date聚合按照日期聚合
  • 2.度量(Metric)聚合
    • Avg Max Min
  • 3.管道(Pipeline)聚合: 根据其他聚合结果再次聚合
GET /index_name/_search
{
    "query": {}, // 限定聚合的范围
    "size": 0,
    "aggs": {
        "聚合后的字段名": { // bucket聚合
            "terms":
                "field": "brand",
                "size": 20 // 希望获取到聚合结果的数量
                "order": {
                    "_count": "asc" // 按照文档数量排序
                }
            }
        },
        // 对聚合后的结果继续聚合
        "aggs": {
            "source_agg": {
                "avg": {
                    "field": "字段名字"
                }
            }
        }
    }
}
