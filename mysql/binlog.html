03.binlog

MySQL binlog

  • MySQL 的 binlog（二进制日志） 是数据库进行 主从复制、数据恢复、增量同步 的关键组件
  • 它记录了所有导致数据变更的 SQL 语句或行级操作。不同的 binlog 格式决定了记录内容的粒度和解析方式。
格式 描述 典型使用场景
STATEMENT 记录执行的 SQL 语句 早期版本默认格式，适用于简单场景
ROW 记录每一行变更的内容（变化前后） 高精度数据同步，推荐使用
MIXED 自动选择使用 STATEMENT 或 ROW 记录（根据语句类型） 兼顾兼容性与安全性
  • STATEMENT 模式：原始 SQL 语句，日志量小，占空间少可读性强（容易理解），非确定性语句会引发主从不一致，如：UUID()、NOW()、RAND()、用户函数
  • ROW 模式（推荐）：每一行变更前后的数据，完整、准确、无歧义，适合数据同步、解析、恢复，日志体积大
  • MIXED 模式：自动在 ROW 和 STATEMENT 之间切换，可安全执行的使用 STATEMENT，含有函数、临时表等复杂情况用 ROW
