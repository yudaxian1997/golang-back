05.ID

分库分表之后，id主键处理

  • 常见主键 ID 生成策略
    • UUID（通用唯一标识符）:不排序，不连续，写入性能差（B+树频繁分裂）
    • 数据库分段（Segment ID）模式: 依赖中心数据库（高可用问题）
    • Snowflake（雪花算法）
      • 0 | 时间戳 | 数据中心 ID | 机器 ID | 序列号

正确做法：业务字段做路由，雪花 ID 仅做主键

推荐方案： 用业务字段（如 user_id 或订单来源）做路由键，用于决定落在哪个库和表； 用雪花 ID 作为主键（唯一 + 全局有序），作为记录的唯一标识； 建立映射关系，便于查询。
