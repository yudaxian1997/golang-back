03.缓存过期

redis的过期策略

  • 当你使用 EXPIRE、TTL 等设置 key 的生命周期时，Redis 有以下三种方式处理过期 key
    • 定时删除（定期）: 每个 key 设置一个过期时间，Redis 会为其创建一个定时任务，到期立即删除。
    • 惰性删除（访问时）: 读取某个 key 时，发现它已过期，此时才删除它。如果一个 key 永远不被访问，可能一直占用内存。
    • 定期随机删除（定期清理）:
      • Redis 每秒钟进行若干次（默认 10 次）过期 key 检查
      • 随机抽取一些 key，如果过期就删除
      • 如果删除比例较高，则继续循环（最多 16 次）
    • Redis 实际使用的是 惰性删除 + 定期随机删除 结合。
  • 内存淘汰机制（内存不足时才触发）

当 Redis 的内存达到配置上限（maxmemory），再新增数据时，就会根据配置的淘汰策略进行清理。

策略 说明
noeviction ❌ 不淘汰（默认），写操作失败，返回错误
allkeys-lru 所有 key 中，淘汰最久未使用的 key
volatile-lru 仅对设置了过期时间的 key 使用 LRU 淘汰
allkeys-random 随机淘汰任意 key
volatile-random 随机淘汰设置了过期时间的 key
volatile-ttl 优先淘汰快过期的 key（TTL 最小）
allkeys-lfu Redis 4.0+，使用 LFU（最少使用）策略
volatile-lfu 对过期 key 使用 LFU 策略
