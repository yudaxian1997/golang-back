04.集群

Redis主从复制

  • 建立连接阶段: 从节点发送PSYNC命令 首次复制使用全量同步 断线重连使用部分重同步
  • 数据同步阶段: 主节点执行BGSAVE生成RDB文件 主节点将RDB文件发送给从节点
  • 命令传播阶段: 主节点将写命令发送给从节点
  • 网络延迟和带宽限制 大量数据同步时的内存压力 避免主从切换时的数据丢失

Redis Cluster的槽位

  • Redis Cluster 将整个键空间 按哈希槽划分成 16384 个 slot，编号 0 ~ 16383
  • Redis 采用 CRC16(key) % 16384 算法，将每个 key 映射到一个 slot。
  • 每个 slot 被分配到某个主节点（master），主节点负责该 slot 范围内的数据。
  • 一个主节点可负责多个 slot，多个主节点共同组成集群。

    Redis 集群扩缩容的常见操作

    扩容（增加节点）
  • 通常将某些 slot 从已有节点迁移到新节点上。
  • 使用 redis-cli --cluster reshard 执行 slot 重分配。
  • Redis 会在后台进行 MIGRATE 命令搬迁 key，同时保持数据一致性。
    缩容（删除节点）
  • 先将该节点上的 slot 迁移给其他节点。
  • 等所有 slot 都迁走后才能安全删除该节点。
